Installing openSmtp.net

Application

The application was designed to work with the Microsoft .Net 1.1 runtime, it must have this installed on the target machine in order to function properly.
  1. Create a new virtual directory, for example 'openSmtp', in your IIS5/dotnet server
  2. set the permissions to include 'scripts only',
  3. Copy everything to the new openSmtp directory
  4. In the IIS admin cosole specify the directory as it's own application.
  5. Also, in the IIS admin cosole specify the web directory as it's own application so the web application may run in its own space.

Nunit tests

The test program (nunit.exe) needs the .net runtime installed. Run Nunit, select openSmtp.dll and then run the tests.
